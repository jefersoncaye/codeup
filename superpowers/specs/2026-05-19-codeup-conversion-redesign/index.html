    CodeUp — Redesign da Home para Página de Conversão¶

    Contexto¶

    O site atual usa MkDocs Material com a home page no estilo documentação (grid de cards técnicos). O objetivo é transformar a home em uma página de conversão focada em cursos — estilo gotasdetecnologia.com.br — mantendo o MkDocs para as páginas de detalhe de cada curso e o blog.

    Objetivo¶

    Página principal = página de vendas dos cursos. Cada curso tem uma página de detalhe com currículo, artigos técnicos (onde existirem) e CTA para o Udemy. Docs genéricas saem do nav principal.

    Arquitetura¶

    Template customizado (overrides/home.html)¶

    A home page usa um template Jinja2 que estende o Material base mas substitui o bloco content por HTML puro de conversão. O docs/index.md tem apenas front matter com template: home.html e hide: [navigation, toc].

    Python para QAs — 8 seções, 32 aulas¶

    • Seção 1 — Introdução: O papel do Python na QA, Instalação e configuração (Windows), Criação do Projeto no GitHub, Executando scripts Python
    • Seção 2 — Fundamentos básicos: Sintaxe básica, Comentários e boas práticas, Tipos de dados primitivos, Operadores aritméticos/relacionais/lógicos, Entrada e saída de dados, Conversão de tipos (casting)
    • Seção 3 — Estruturas de Controle: Estruturas condicionais (if, elif, else), Estruturas de repetição (for, while), Laços aninhados
    • Seção 4 — Estruturas de Dados: Listas, Tuplas, Dicionários, Métodos e operações, Iteração sobre estruturas
    • Seção 5 — Funções: Criação e uso de funções, Valores padrão e argumentos nomeados, args e *kwargs
    • Seção 6 — Manipulação de Strings e Datas: Strings (operações e métodos), Formatação e f-strings, Datas e Horas com datetime, Formatação e Comparação de Datas
    • Seção 7 — Manipulação de Arquivos: Caminhos com pathlib, Escrita e Append, Leitura, Manipulação de JSON
    • Seção 8 — POO: Introdução a POO, Métodos e Encapsulamento, Herança e Polimorfismo

    Automação Web com Playwright — 14 seções, 69 aulas¶

    • Seção 1 — Setup e Primeiros Passos: Recursos, Instalação Python+PyCharm, Ambiente Virtual, Instalando Playwright+Pytest, Primeiro Teste, Estrutura Básica de Projeto
    • Seção 2 — Extra: Python Básico: Sintaxe, Tipos, Entrada/Saída, Casting, Condicionais, Repetição, Listas, Dicionários, Funções, POO, Encapsulamento
    • Seção 3 — Fundamentos do Playwright: Seletores (get_by_role, get_by_text, get_by_placeholder, get_by_label, get_by_title, locator), Comandos (click, fill, check, select_option, press, type, hover, dblclick, expect), Auto-wait
    • Seção 4 — Page Object Model: Introdução e Estrutura Básica, Criando Página e Teste de Login
    • Seção 5 — Pytest Fixtures: Criando fixtures contexto e page, Escopos de Fixtures
    • Seção 6 — Exercícios e Boas Práticas: Criar Conta, Login/Logout, Adicionar Produto ao Carrinho
    • Seção 7 — Uploads e Downloads: Upload de arquivos, Download de Arquivos
    • Seção 8 — Extra: iFrames: Trabalhando com IFrames
    • Seção 9 — Extra: Lógica Condicional: Funções/Propriedades de Locator, Exercícios com IF/Else e While
    • Seção 10 — Extra: Múltiplas Janelas e Abas: Novas Abas, Novas Janelas
    • Seção 11 — Execução Avançada e Relatórios: Execução Paralela, Relatórios HTML, Comandos CLI
    • Seção 12 — Autenticação: Autenticação com Storage State
    • Seção 13 — Integração Contínua: Jenkins (Instalação, Criando Job, Publicando Relatório e Artefatos)
    • Seção 14 — Extra: Utilitários: Utilitário para Comparação de Arquivos

    Automação de API com Playwright — 6 seções, 20 aulas¶

    • Seção 1 — Fundamentos essenciais: Introdução ao conceito de API, Simulado Métodos HTTP, Preparando o ambiente, Primeiro teste de API na prática
    • Seção 2 — Primeiros passos com Playwright para APIs: Requisições e respostas, CRUD completo de usuários, Autenticação e Tokens
    • Seção 3 — Arquitetura com OOP: Introdução + Estrutura do projeto, Criando classes de serviços de API, Helpers e reutilização de código, Simulado Estrutura e Boas Práticas
    • Seção 4 — Garantia de Qualidade e JSON Schema: Validação de mensagens de erro, Validação usando JSON Schema
    • Seção 5 — Testes orientados a dados (DDT): Testes orientados a dados
    • Seção 6 — Relatórios e Performance: Relatórios com pytest-html e Allure, Execução paralela com pytest-xdist

    Automação com TestComplete — 8 seções, 35 aulas¶

    • Seção 1 — Fundamentos e IDE: O que é TestComplete e como funciona, Instalação e Primeiro Teste, Conhecendo o Ambiente
    • Seção 2 — NameMapping: Introdução ao NameMapping, Aliases vs Full Name, Configurações, Coringas nos Mapeamentos, Boas Práticas
    • Seção 3 — Operações com Keywords: Keywords (On-Screen Action, Menu Action), If-Else, While Loop, For Loop, Run Test, Set Variable Value, Delays
    • Seção 4 — Checkpoints: Property, File, Table, OCR e Region Checkpoints
    • Seção 5 — Parametrização de Keywords: Parametrização, Boas Práticas e Teste Completo
    • Seção 6 — Testes Web: Testes Web com TestComplete
    • Seção 7 — Data-Driven Testing: DDT com Arquivo CSV
    • Seção 8 — Configurações: Playback, Visualizer

    • Seção 2 — Tipos de Testes de Performance: Smoke, Load, Stress, Spike, Soak, Breakpoint + Simulado
    • Seção 3 — Fundamentos Essenciais: Ciclo de Vida no k6, Trabalhando com options

    Features a remover do mkdocs.yml¶

    • navigation.tabs — com a home customizada e sem abas genéricas, tabs não fazem sentido
    • navigation.tabs.sticky — dependente do anterior
    • navigation.sections — cursos não precisam de agrupamento expandível
    • navigation.indexes — substituído pela estrutura de detalhe

    Features a manter¶

    navigation.top, search.highlight, search.share, search.suggest, content.code.copy, content.code.annotate, content.tabs.link
